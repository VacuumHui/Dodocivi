# Технический аудит исходного проекта

## Выявленные проблемы

1. **Монолитный `main.dart`.** UI, сеть, JSON-модели, файловое хранилище и экспорт были смешаны в одном StatefulWidget.
2. **Небезопасный разбор API.** Код напрямую обращался к динамическим полям ответа Civitai и мог падать на неполных или изменённых записях.
3. **Нет тайм-аута и повторной попытки.** При зависшем или недоступном API экран мог оставаться в неопределённом состоянии.
4. **Только первые 50 изображений.** Пагинация Civitai не использовалась, после чего приложение считало ленту законченной.
5. **Возможные `setState` после закрытия экрана.** Асинхронные операции не всегда проверяли `mounted`.
6. **Тихая потеря данных.** Повреждённый JSON перехватывался пустым `catch`, затем мог быть перезаписан новым массивом.
7. **Нет защиты от дублей.** Один и тот же источник можно было сохранить многократно.
8. **Внутренние данные совпадали с экспортом.** Нельзя было хранить технические поля, не загрязняя обучающий JSON.
9. **Устаревший API экспорта.** Использовался старый статический вызов `Share.shareXFiles`.
10. **Хрупкая GitHub-сборка.** Workflow брал произвольную текущую stable-версию Flutter, не запускал анализ и тесты, а изменение `local.properties` не гарантировало нужный `minSdk`.
11. **Отсутствовала полноценная Android-конфигурация.** Репозиторий зависел от результата `flutter create .` без детерминированной постнастройки.
12. **Слабый мобильный UX.** Квадратное кадрирование всех изображений, нет истории, удаления, прогресса, метаданных, состояния догрузки и адаптации к широкому экрану.

## Что исправлено

- Код разделён на модели, сервисы, экраны и переиспользуемые виджеты.
- Добавлены типизированный разбор Civitai, фильтрация повреждённых записей, тайм-ауты и пользовательские ошибки.
- Реализованы курсорная пагинация, пропуск пустых страниц, устранение дублей и предварительная загрузка изображений.
- Добавлено атомарное локальное сохранение через временный файл.
- Добавлена однократная миграция старого `sdxl_dataset.json` во внутренний формат версии 1.1.
- Экспорт снова создаёт чистый массив только с `instruction`, `input`, `output`.
- Добавлены история датасета, удаление отдельных записей, полная очистка и счётчик.
- Интерфейс переведён на Material 3 и адаптирован для смартфонов и широких экранов.
- Обновлён `share_plus` и его API.
- Сохранён прежний Android application ID `com.example.sdxl_collector`, поэтому идентификатор приложения не меняется относительно исходной сборки.
- GitHub Actions закреплён на конкретной версии Flutter и Java 17; сборка включает форматирование, анализ, тесты, release APK и SHA-256.
- Скрипт Android-настройки фиксирует `minSdk`, AGP, Kotlin, Gradle, INTERNET permission и название приложения.
